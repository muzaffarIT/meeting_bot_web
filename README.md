# Newton Academy Bot + Admin Panel (v2)

Это уже **вторая версия каркаса** проекта:
- Telegram-бот для родителей
- Google Sheets как база данных
- веб-панель для менеджеров и руководителя
- напоминания по Ташкенту (`Asia/Tashkent`)
- авторизация сотрудников **не по Telegram ID**, а по **логину и паролю**

## Что реализовано

### Бот для родителей
- вход по персональной ссылке (`deep link`)
- первый экран: выбор языка
- до подтверждения встреча **без локации**
- кнопка `✅ Подтверждаю встречу`
- после подтверждения бот показывает дату, время, адрес и кнопку `📍 Открыть локацию`
- кнопка связи с менеджером:
  - сначала через `@telegram`, если он указан у сотрудника
  - если Telegram username не заполнен, используется телефон

### Напоминания
Бот отправляет напоминания в **ташкентском времени**:
- за **3 дня**
- за **1 день**
- за **6 часов**
- за **3 часа**
- за **2 часа**

Логика не ломается, если встреча создана поздно:
- если запись создана за 2 дня, этап `3 дня` просто пропускается
- если запись создана за 5 часов, этапы `3 дня`, `1 день`, `6 часов` уже пропустятся, но система всё равно сможет отправить `3 часа` и `2 часа`, если они ещё впереди

### Веб-панель
Есть две роли:
- `manager` — видит только своих лидов
- `admin` / `owner` — видит всех лидов и общую статистику

Функции панели:
- вход по логину/паролю
- список лидов
- создание нового лида
- карточка лида
- редактирование даты, времени, статуса, полей `пришел` и `купил`
- общая статистика по менеджерам
- автоматическая генерация персональной ссылки в бота

## Почему авторизация не по Telegram ID

Ты правильно заметил: если лидорубов много и они меняются, привязка только к Telegram ID неудобна.

В этой версии я предлагаю **более управляемую схему**:

### Сотрудники авторизуются через:
- `логин`
- `пароль`
- `роль`

Их данные хранятся в листе `users` в Google Sheets:
- логин
- ФИО
- роль
- телефон
- Telegram username
- активность
- salt
- hash пароля

### Почему это лучше
- сотрудника легко добавить / отключить
- не нужно ловить его Telegram ID
- можно быстро сменить пароль
- можно перевести сотрудника из `manager` в `admin`
- один сотрудник может не пользоваться ботом, но работать в панели

## Как менеджер связывается с родителем и наоборот

### Родитель → менеджер
В боте кнопка связи с менеджером строится так:
1. если у менеджера есть `Telegram username`, бот даёт ссылку `https://t.me/username`
2. если Telegram username нет, используется `tel:+998...`

### Менеджер → родитель
Пока это не автоматизированный чат внутри панели. В MVP менеджер видит:
- имя родителя
- телефон
- статус
- персональную ссылку в бот

Дальше он может:
- отправить ссылку вручную в Telegram/WhatsApp
- позвонить
- напомнить дополнительно

## Структура проекта

### `bot.py`
Основной Telegram-бот.

### `reminder_worker.py`
Фоновый воркер, который каждые `POLL_INTERVAL_SECONDS` секунд проверяет, кому пора отправить напоминание.

### `app.py`
Веб-панель на FastAPI.

### `sheets.py`
Вся работа с Google Sheets.

### `auth_utils.py`
Хеширование паролей через `PBKDF2-HMAC-SHA256`.

### `constants.py`
Русские заголовки таблиц, роли и статусы.

### `texts.py`
Тексты бота и напоминаний.

### `utils.py`
Таймзона, булевы флаги, разбор даты встречи, генерация ID.

### `init_sheet.py`
Создаёт и заполняет заголовки листов `leads` и `users`.

### `create_user.py`
Создаёт сотрудника для входа в панель.

### `create_test_lead.py`
Создаёт тестовый лид и печатает персональную ссылку в бота.

## Подготовка на MacBook

### 1. Создай проект и зайди в папку
```bash
cd /path/to/newton_bot_v2
```

### 2. Виртуальное окружение
```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 3. Установка зависимостей
```bash
pip install -r requirements.txt
```

### 4. Настрой `.env`
```bash
cp .env.example .env
```

Заполни:
- `BOT_TOKEN`
- `BOT_USERNAME`
- `GOOGLE_SHEET_NAME`
- `SERVICE_ACCOUNT_FILE`
- `SESSION_SECRET`
- `DEFAULT_BRANCH_ADDRESS`
- `DEFAULT_BRANCH_LOCATION_URL

### 5. Положи `service_account.json`
В корень проекта.

### 6. Расшарь Google Sheets
Открой таблицу и дай доступ email сервисного аккаунта.

### 7. Инициализация листов
```bash
python3 init_sheet.py
```

### 8. Создай первого пользователя
```bash
python3 create_user.py
```

Рекомендуется сначала создать `owner`.

### 9. Запусти админ-панель
```bash
python3 app.py
```

Откроется на:
```text
http://127.0.0.1:8000
```

### 10. Запусти бота
Во втором терминале:
```bash
source .venv/bin/activate
python3 bot.py
```

### 11. Запусти воркер напоминаний
В третьем терминале:
```bash
source .venv/bin/activate
python3 reminder_worker.py
```

## Как выглядит рабочий процесс

### 1. Руководитель создаёт сотрудников
Через `create_user.py`.

### 2. Менеджер заходит в панель
Использует свой логин и пароль.

### 3. Менеджер создаёт лида
В панели `/leads/new`.

### 4. Система генерирует deep-link
В карточке лида есть персональная ссылка вида:
```text
https://t.me/<bot_username>?start=<lead_id>
```

### 5. Менеджер отправляет ссылку родителю
Сообщением вручную.

### 6. Родитель заходит в бота
- выбирает язык
- видит просьбу подтвердить встречу
- только после подтверждения получает локацию

### 7. Воркер шлёт напоминания
По расписанию и по Ташкенту.
